🔒 Fix ambiguous id reference in video update policy

🎯 What: Fixed a logic error in the video status update policy where `id = id` was ambiguous.
⚠️ Risk: An ambiguous column reference could allow users to bypass row level security policies and update attributes they shouldn't be able to (like status), leading to unauthorized modification of video status (e.g. unblocking a blocked video or bypassing publishing workflows).
🛡️ Solution: Added table aliases to explicitly disambiguate the comparison between the row being checked and the overall `videos.id`. Now it accurately checks that the status of the current row being updated matches its existing status in the database.
